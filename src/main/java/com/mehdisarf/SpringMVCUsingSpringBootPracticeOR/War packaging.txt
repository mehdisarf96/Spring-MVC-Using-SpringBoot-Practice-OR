chetor Spring Boot ro config konim tori ke betunim package konim yek War file ro bejaye
the Default executable jar.

yad gereftim ke chejur app e mun ro config konim to be packaged as an executable jar
so that can be ran within a main method or deploy to a servlet container.

This all starts in the pom.xml file !

we need to modify the <packaging></<packaging> from jar to war:

    <groupId>com.mehdisarf</groupId>
    <artifactId>Spring-MVC-Using-SpringBoot-Practice-OR</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <packaging>war</packaging>

and the next thing we need to do is specify one of our dependency as 'provided':
be in shekl ke ye dependency jadid be in shekl ezafe mikonam:

	<dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-tomcat</artifactId>
            <scope>provided</scope>
        </dependency>

va in neshun mide ke We do not need to include any of the libraries associated with
the spring-boot-starter-tomcat dependency because we expect them to be 'provided'!
this is going to allow our packaged application to run as WAR on standalone tomcat server.

-----------------------------------------------------

ba in taqirati ke dar pom.xml dadim, hala mitunim berim be class ii ke @SpringBootApplication dare.

this is the root class that contains our main method va taqirati ke inja mikham bedam kheyli sade and.

tanha kari ke lazeme tu in class anjam bedam ine ke bayad in class
extends kone the SpringBootServletInitializer class.


@SpringBootApplication
public class SpringMvcUsingSpringBootPracticeOrApplication
        extends SpringBootServletInitializer {

    public static void main(String[] args) {
        SpringApplication.run(SpringMvcUsingSpringBootPracticeOrApplication.class, args);
    }
}


berim tuye SpringBootServletInitializer bash ashna shim.

mibiniim ke SpringBootServletInitializer implement mikone interface e WebApplicationInitializer ro.

Doc:
An opinionated WebApplicationInitializer to run a SpringApplication
from a traditional WAR deployment

and this is the class that has the hooks into the Servlet 3-0 Container
and we've used it in the past to configure the Dispatcher Servlet.

akharin kar ham ine ke age tu eclipce hasti, yek execution enviroment add koni (masalan java 8)
alan dg mitini app e mun ro run konim va Ok.

Hala dar in lahze, We are going to Package the application as WAR.

Finally, we tell maven to package the project in to WAR.
In your pom.xml, change the attribute value for packaging from jar to war

<packaging>war</packaging>

Then try to build your project using: 'mvn package'
